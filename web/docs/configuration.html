配置指南

自定义Lebit.sh以满足您的需求

配置概述

Lebit.sh提供了灵活的配置选项，允许您根据自己的需求和偏好自定义工具行为。本文档将介绍如何配置Lebit.sh的各个方面，包括全局设置和模块特定设置。

配置文件位置

Lebit.sh的配置文件存储在以下位置：

  • 全局配置: ~/.config/lebit/config
  • 模块配置: ~/.config/lebit/modules/<module_name>/config
  • 用户数据: ~/.local/share/lebit/

安装Lebit.sh后，这些目录会自动创建，并填充默认配置文件。

全局配置

全局配置文件控制Lebit.sh的整体行为和外观。以下是一些重要的全局配置选项：

# Lebit.sh 全局配置 # 界面设置 COLOR_SCHEME="default" # 可选: default, dark, light, custom ENABLE_ANIMATIONS=true # 是否启用动画效果 VERBOSE_OUTPUT=false # 是否显示详细输出 # 一般设置 DEFAULT_MODULE="system" # 启动时默认打开的模块 CHECK_UPDATES=true # 是否自动检查更新 UPDATE_INTERVAL=7 # 检查更新的时间间隔（天） # 系统设置 BACKUP_BEFORE_CHANGES=true # 在进行更改前是否创建备份 MAX_BACKUP_COUNT=5 # 保留的最大备份数量 BACKUP_LOCATION="~/.local/share/lebit/backups"

要修改这些设置，只需编辑配置文件并根据需要更改值：

nano ~/.config/lebit/config

模块配置

每个模块都有自己的配置文件，允许您自定义该模块的特定行为。以下是一些常用模块的配置示例：

系统模块配置

# 系统模块配置 # 清理设置 CLEAN_PACKAGE_CACHE=true # 清理软件包缓存 CLEAN_LOGS=true # 清理旧日志文件 CLEAN_TEMP_FILES=true # 清理临时文件 LOG_RETENTION_DAYS=30 # 保留日志的天数 # 优化设置 OPTIMIZE_SWAP=true # 优化交换分区使用 OPTIMIZE_FILESYSTEM=true # 优化文件系统 OPTIMIZE_CPU=true # 优化CPU性能

Docker模块配置

# Docker模块配置 # 安装设置 DOCKER_VERSION="latest" # Docker版本 DOCKER_COMPOSE_VERSION="latest" # Docker Compose版本 INSTALL_METHOD="official" # 安装方法: official, package, manual # 网络设置 DEFAULT_NETWORK_DRIVER="bridge" CUSTOM_DNS_SERVERS="8.8.8.8,1.1.1.1" # 存储设置 DEFAULT_STORAGE_DRIVER="overlay2" DOCKER_ROOT="/var/lib/docker"

使用环境变量

除了配置文件外，您还可以使用环境变量来临时覆盖配置设置。这对于测试或在特定情况下修改行为非常有用。

Lebit.sh遵循以下环境变量命名约定：

LEBIT_[MODULE]_[SETTING]

例如，要临时禁用Docker模块的自动更新检查，可以使用：

LEBIT_DOCKER_CHECK_UPDATES=false lebit docker

通过命令行配置

Lebit.sh提供了一个配置命令，允许您直接从命令行查看和修改设置：

查看当前配置

lebit config list

查看特定模块的配置

lebit config list docker

修改配置设置

lebit config set BACKUP_BEFORE_CHANGES false

修改模块特定设置

lebit config set docker DOCKER_VERSION 20.10.8

重置为默认配置

lebit config reset

配置文件语法

Lebit.sh配置文件使用简单的键值对语法。以下是一些语法规则：

  • 每行一个键值对，格式为：KEY=VALUE
  • 注释以#开头
  • 字符串值可以用引号括起来，也可以不用
  • 布尔值用true或false表示
  • 数组用逗号分隔，例如：ARRAY_VALUE="item1,item2,item3"

高级配置

对于高级用户，Lebit.sh支持更复杂的配置选项和功能：

创建配置文件夹

您可以将配置分离为多个文件，存储在特定目录中：

~/.config/lebit/conf.d/ 01-base.conf 02-modules.conf 03-custom.conf

Lebit.sh会按照字母数字顺序加载这些文件，后面的设置会覆盖前面的设置。

配置文件模板

您可以创建配置模板，以便在多个系统上快速部署：

lebit config export template.conf

然后在另一个系统上导入：

lebit config import template.conf
安全提示

配置文件可能包含敏感信息，如API密钥或凭据。确保将配置文件的权限设置为仅所有者可读写：

chmod 600 ~/.config/lebit/config

配置示例

以下是一些常见场景的配置示例：

服务器优化配置

# Lebit.sh 服务器优化配置 DEFAULT_MODULE="system" BACKUP_BEFORE_CHANGES=true BACKUP_LOCATION="/opt/backups/lebit" # 系统模块设置 OPTIMIZE_SWAP=true SWAP_SWAPPINESS=10 OPTIMIZE_FILESYSTEM=true OPTIMIZE_CPU=true CPU_GOVERNOR="performance" # Docker模块设置 DOCKER_VERSION="stable" DEFAULT_STORAGE_DRIVER="overlay2" ENABLE_LIVE_RESTORE=true

开发环境配置

# Lebit.sh 开发环境配置 DEFAULT_MODULE="dev" COLOR_SCHEME="dark" ENABLE_ANIMATIONS=true # 开发模块设置 DEV_DEFAULT_EDITOR="vscode" DEV_INSTALL_GIT=true DEV_GIT_USER_NAME="Your Name" DEV_GIT_USER_EMAIL="your.email@example.com" DEV_NODEJS_VERSION="16" DEV_PYTHON_VERSION="3.9"

故障排除

如果您遇到配置问题，可以尝试以下方法：

  • 运行诊断命令：lebit config validate
  • 检查配置语法错误：lebit config check
  • 以调试模式运行Lebit.sh：LEBIT_DEBUG=true lebit
  • 重置为默认配置：lebit config reset
