Configuration Guide

Lebit.sh is designed to be highly customizable, allowing you to tailor it to your specific needs. This guide explains how to configure various aspects of Lebit.sh, from basic settings to module-specific configurations.

Configuration File

The main configuration file for Lebit.sh is located at ~/.config/lebit/config.yaml. This YAML file contains all the settings that control how Lebit.sh behaves.

If the file doesn't exist, it will be created with default values when you run Lebit.sh for the first time. You can edit this file directly with any text editor, or use the configuration commands provided by Lebit.sh.

Using Configuration Commands

Lebit.sh provides a set of commands for managing configuration settings without having to edit the configuration file directly:

# View all current configuration settings
lebit config list

# Get the value of a specific setting
lebit config get [key]

# Set the value of a specific setting
lebit config set [key] [value]

# Reset a specific setting to its default value
lebit config reset [key]

# Reset all settings to their default values
lebit config reset --all

Global Configuration Options

These settings apply to Lebit.sh as a whole:

Setting Description Default Value Example
install.path The directory where Lebit.sh is installed /usr/local/lebit lebit config set install.path /opt/lebit
log.level The logging level (debug, info, warn, error) info lebit config set log.level debug
update.check Whether to automatically check for updates true lebit config set update.check false
update.interval How often to check for updates (in days) 7 lebit config set update.interval 14
proxy.enabled Whether to use a proxy for network operations false lebit config set proxy.enabled true
proxy.url The URL of the proxy server "" lebit config set proxy.url http://proxy.example.com:8080

Module-Specific Configuration

Each module has its own set of configuration options. These are organized under the module name in the configuration file.

System Module Configuration

Setting Description Default Value Example
system.clean.logs Whether to clean system logs during cleanup true lebit config set system.clean.logs false
system.clean.temp Whether to clean temporary files during cleanup true lebit config set system.clean.temp false
system.clean.cache Whether to clean package caches during cleanup true lebit config set system.clean.cache false
system.optimize.cpu Whether to optimize CPU settings true lebit config set system.optimize.cpu false
system.optimize.memory Whether to optimize memory settings true lebit config set system.optimize.memory false
system.optimize.disk Whether to optimize disk I/O settings true lebit config set system.optimize.disk false

Docker Module Configuration

Setting Description Default Value Example
docker.data.path The directory where Docker stores its data /var/lib/docker lebit config set docker.data.path /opt/docker
docker.compose.version The version of Docker Compose to install latest lebit config set docker.compose.version 2.5.0
docker.registry The Docker registry to use docker.io lebit config set docker.registry registry.example.com
docker.prune.interval How often to automatically prune Docker resources (in days, 0 to disable) 0 lebit config set docker.prune.interval 7

Configuration Best Practices

Here are some recommendations for configuring Lebit.sh:

  • Use Version Control: Consider keeping your configuration file in a version control system like Git to track changes and easily revert if needed.
  • Backup Your Configuration: Regularly back up your configuration file, especially before making significant changes.
  • Document Your Changes: Add comments to your configuration file explaining why you made certain changes, which can be helpful for future reference.
  • Test Changes in a Safe Environment: If possible, test configuration changes in a non-production environment before applying them to critical systems.

Advanced Configuration

For advanced users, Lebit.sh supports creating custom modules and extending the functionality of existing modules. To learn more about advanced configuration options, refer to the Advanced Configuration Guide in our wiki.
