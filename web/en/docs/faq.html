What are the system requirements for Lebit.sh?

Lebit.sh is designed to work on most Linux distributions, including Ubuntu, Debian, CentOS, Fedora, and Arch Linux. The minimum requirements are:

  • Linux kernel 4.x or newer
  • Bash shell
  • curl or wget
  • At least 100MB of free disk space
  • Internet connection for installation and updates
How do I install Lebit.sh without using curl?

If you prefer not to use curl or if it's not available on your system, you can install Lebit.sh using wget:

wget -O - https://lebit.sh | sh

Alternatively, you can manually download the installation script and run it:

wget https://lebit.sh/install.sh
chmod +x install.sh
./install.sh
Is it safe to pipe the installation script directly to shell?

We understand the concerns about piping scripts directly to shell. For those who prefer to review the installation script before running it, you can download the script first and inspect it:

curl -o install.sh https://lebit.sh/install
less install.sh  # Review the script
chmod +x install.sh
./install.sh

Our installation script is open source, and we encourage users to review it before running. The script is designed to be transparent and does not perform any actions without user confirmation.

How do I update Lebit.sh to the latest version?

Updating Lebit.sh is simple. Just run the following command:

lebit update

This will check for updates and, if available, download and install the latest version of Lebit.sh. Your configuration and installed modules will be preserved during the update.

Can I use Lebit.sh in scripts or automated workflows?

Yes, Lebit.sh is designed to work well in scripts and automated workflows. Most commands support a --quiet or -q flag to suppress interactive prompts and output, making them suitable for scripting:

lebit install docker --quiet

You can also use the --yes or -y flag to automatically confirm all prompts:

lebit install docker --yes

For exit codes, Lebit.sh follows the standard convention of returning 0 for success and non-zero values for errors, allowing you to check the outcome in your scripts.

How do I uninstall Lebit.sh completely?

To uninstall Lebit.sh and remove all its files from your system, run:

lebit uninstall --purge

This will remove Lebit.sh, its configuration files, and all installed modules. If you want to keep your configuration for future reinstallation, omit the --purge flag:

lebit uninstall

Note that this will not uninstall applications or services that were installed using Lebit.sh modules. You'll need to uninstall those separately.

I'm getting "Permission denied" errors when running Lebit.sh commands

This typically occurs when you don't have the necessary permissions to perform the requested action. There are a few ways to address this:

  1. Run the command with sudo:

    sudo lebit [command]

  2. Check if you have write permissions to the directories Lebit.sh is trying to access. For example, if you've configured a custom installation path, make sure you have write permissions to that directory.

  3. If you're encountering permission issues with a specific module, check the module's documentation for any special permission requirements.

Lebit.sh says "Command not found" after installation

If you're seeing a "Command not found" error when trying to run Lebit.sh, it's likely that the installation directory is not in your PATH. Here's how to fix this:

  1. Check where Lebit.sh is installed:

    find / -name lebit 2>/dev/null

  2. Add the directory to your PATH by adding the following line to your ~/.bashrc or ~/.zshrc file:

    export PATH=$PATH:/path/to/lebit/directory

  3. Reload your shell configuration:

    source ~/.bashrc

    or

    source ~/.zshrc

Alternatively, you can run Lebit.sh using the full path to the executable:

/path/to/lebit/directory/lebit [command]
How do I diagnose issues with Lebit.sh?

If you're experiencing issues with Lebit.sh, you can use the built-in diagnostic tools to help identify the problem:

  1. Enable debug logging:

    lebit config set log.level debug

  2. Run the command that's causing issues with the --verbose flag:

    lebit [command] --verbose

  3. Check the log file for detailed information:

    cat ~/.config/lebit/logs/lebit.log

  4. Run the diagnostics command to check for common issues:

    lebit diagnostics

If you're still having issues, you can open an issue on our GitHub repository with the output of these commands.

Can I create my own custom modules for Lebit.sh?

Yes, Lebit.sh supports custom modules. To create a custom module:

  1. Create a new directory in ~/.config/lebit/modules/ with your module name:

    mkdir -p ~/.config/lebit/modules/mymodule

  2. Create a main.sh file in this directory, which will be the entry point for your module:

    touch ~/.config/lebit/modules/mymodule/main.sh

  3. Implement your module following the Lebit.sh module API. Here's a simple example:

    #!/bin/bash

# Main function for the module
mymodule_main() {
    echo "Hello from my custom module!"
}

# Initialize the module
mymodule_init() {
    echo "Initializing my custom module..."
}

# Clean up function
mymodule_cleanup() {
    echo "Cleaning up my custom module..."
}

# Add any additional functions here
mymodule_custom_function() {
    echo "This is a custom function!"
}

  4. Make the script executable:

    chmod +x ~/.config/lebit/modules/mymodule/main.sh

After creating your module, you can use it with Lebit.sh commands:

lebit mymodule

For more detailed information on creating custom modules, refer to the Custom Modules Guide in our wiki.

Why do some modules require sudo privileges?

Some modules require sudo privileges because they need to perform actions that require elevated permissions, such as:

  • Installing system packages
  • Modifying system configuration files
  • Managing system services
  • Installing software in system directories

Lebit.sh will always inform you when a command requires sudo privileges and will prompt for your password when needed. If you prefer not to enter your password for each command, you can run Lebit.sh with sudo from the beginning:

sudo lebit [command]

However, for security reasons, we recommend only using sudo when necessary and responding to the prompts as they appear.

Can I use Lebit.sh on non-Linux systems like macOS or Windows?

Lebit.sh is primarily designed for Linux systems. However:

  • macOS: Some modules may work on macOS, especially those that don't rely on Linux-specific features. However, full compatibility is not guaranteed, and you may encounter issues with certain modules.

  • Windows: Lebit.sh is not designed to work natively on Windows. However, you can use it within the Windows Subsystem for Linux (WSL) if you have that installed.

We're considering expanding Lebit.sh to better support other operating systems in the future. If you're interested in helping with this effort, please check our contribution guidelines.
